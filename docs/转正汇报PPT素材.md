# 工作汇报 · PPT 方向稿（面向领导）

> **用途**：定汇报叙事与页序，不是技术说明。抬升一层：**整站智能体平台**在科研场景里解决什么问题、坚持什么方法、取得什么可感知效果。  
> **保留的少量架构词**（必要时口头一带）：语义路由、流式推送（SSE）、工具注册、子进程隔离。

---

## 一、汇报开场：我们是什么、为谁服务

- **定位**：面向生命科学与相关实验数据的**智能体工作台**——把「问、选流程、调参数、跑工具、看结果」串成闭环，而不是单点聊天框。
- **服务对象**：课题组、平台用户、运维与二次开发方；价值锚点是**可交付的分析/工具结果**，不是模型话术量。
- **一句话**：在**可控、可扩展、可运维**的前提下，缩短从科研问题到可验收产出路径。

---

## 二、系统级矛盾（不讲某次迭代，讲长期结构问题）

| 矛盾 | 业务表现 |
|------|----------|
| **意图与执行混谈** | 用户要「干活」，系统却闲聊、空规划或走错分支。 |
| **能力爆炸与边界模糊** | 工具、工作流、插件、外部算力并存，易误触重算或误承诺不存在的能力。 |
| **体验与长链路不同步** | 流式交互一旦异常，界面「假死」，用户信任崩塌。 |
| **规模化接入不可复制** | 新能力若依赖个人经验拼装，质量与上线节奏不可控。 |

---

## 三、总体方法论（平台级回答：我们怎么建这个系统）

1. **分层解耦**：把「判去哪」与「怎么编排」「怎么执行」拆开；判错宁可澄清，也不误进重任务。  
2. **确定性通道**：对已签约、可独立交付的原子能力，保留**不经过冗长规划**的直达路径（与语义判定的快车道互为补充）。  
3. **算力与主链路隔离**：重依赖、重 CPU 的环节**不进主 API 进程**，保障会话与推送稳定。  
4. **契约化交付**：工具与工作流向 UI 输出**约定形态**（可读摘要、图表、结构化附件），避免「一坨 JSON」。  
5. **工程化扩容**：用文档化 SOP 把「能接进来」变成「能重复接、能验收、能上线」。

---

## 四、能力全景（领导听得懂的「几块板」）

- **理解与分流**：结合用户表述、数据是否就绪、外部开关状态与简短上下文，做**结构化路由**（任务 / 说明 / 澄清 / 专项通道等），并配套置信与兜底策略。  
- **编排与执行**：工作流（SOP）与**原子工具注册中心**并存；参数由 schema 约束，便于诊断推荐与执行器对齐。  
- **领域纵深**：组学流水线与**化学/电化学等专业工具**同平台承载；专业包通过**子进程或独立算力**落地，主服务保持轻量。  
- **多形态技能**：系统预置、用户插件、纯引导类能力分层治理，避免「广场有、路由没有」的假能力。  
- **人机协同界面**：流式反馈、过程日志、参数表与推荐应用、结果侧栏展示——**状态可恢复、结果可阅读**。

---

## 五、效果与价值（用领导语言收束）

- **效率**：减少无效对话与误执行，单点工具可**直达结果**，流水线场景减少反复对齐。  
- **稳定**：主链路不因单个重工具拖垮；异常连接下界面仍可恢复操作。  
- **可信**：路由与执行边界清晰，「能做什么」可对用户与审计交代。  
- **可复制**：新能力接入有检查清单与验收路径，团队产能不依赖单人记忆。  
- **可演进**：路由规则、工具清单、工作流域可独立迭代，整体架构不被绑死。

---

## 六、风险与底线（简略带过，显成熟度）

- 数据未就绪时**强制澄清**；无权限的外部算力**不自动放行**。  
- 工具参数命名与返回结构**统一规范**，降低自动化诊断与前端展示错位风险。  
- 前端大文件**微创修改**、编码与布局约束，避免「一改全崩」的维护债。

---

## 七、建议 PPT 页序（可直接当目录用）

1. **背景**：科研场景下「分析平台 + 大模型」的机会与常见失败形态。  
2. **目标**：我们要建的**不是聊天机器人**，而是**可交付的智能体工作台**（一句话 + 示意图占位）。  
3. **矛盾归纳**：意图混谈、能力边界、体验假死、规模化不可复制（四条以内）。  
4. **总体架构**：分层（理解 → 编排 → 执行 → 展现）+ 算力外置原则（一图胜千言）。  
5. **关键方法**：语义分流、确定性专项通道、工具注册与契约、子进程/Worker 隔离。  
6. **能力版图**：组学 / 化学电化学 / 插件与广场（用业务名词，少用文件名）。  
7. **体验与可信**：流式终态与解锁、会话连续性、结果可读。  
8. **工程与规范**：扩容 SOP、评测与验收、镜像与环境底线。  
9. **阶段成效**：稳定性、接入效率、用户侧可感知指标（用你们真实数据填空）。  
10. **下一步**：路由与工具视野持续对齐、领域工具纵深、观测与运营（各 1 条即可）。
