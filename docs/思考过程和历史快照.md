# 思考过程与历史快照 — 复刻与借鉴指南

本文将 **`docs/FRONTEND_THOUGHT_RENDERING_REPLAY.md`**（前端思考区渲染状态机）与 **`docs/METHODOLOGY_Streaming_Thinking_And_Panshi_Tools.md`**（流式与协议方法论）**有机整合**，并补充 **`docs/STATE_SNAPSHOT_AUDIT_AND_SCHEMA_DRAFT.md`** 中与「历史快照 / 时光机」强相关的结论，作为后续同类功能的**单一入口文档**。

- 更细的 DOM 行号与专项排障，仍可回查上述三份原文。  
- 修改 `stream_utils`、`orchestrator._emit_sse`、前端 `handleServerEvent` 时，请同步校对本节并更新文末维护记录。

### 落地状态（与仓库同步，执行记录 / 快照 / 历史）

| 能力 | 后端 `orchestrator._emit_sse` | 前端历史 `createRestoreAIMessageRowWithSnapshot`（`index.html`） |
|------|------------------------------|------------------------------------------------------------------|
| 思考区 | `thought` → 累加 `reasoning` | 有 `snap.reasoning` 则挂载 `.agent-thought-block` |
| 执行记录 | `status`（非 AgenticLog）→ `process_log[]`；`done` 追加完成行并写 `duration` | 有 `snap.process_log` 则 `renderProcessLogToLogZone`；否则 checklist |
| 正文 | `message` → 累加 `text` | `textZone` + `extractAndStripSuggestionTags` |
| 计时器 | `done` 写 `duration` | footer `.generation-timer` |

**改架构时的四联检查**：动 SSE 语义 / `_emit_sse` 聚合 / `server` 入库字段 / 历史恢复函数之一，须评估其余三处是否同步（见 **`.cursor/rules/gibh-architecture-constitution.mdc` §5**）。

## 一、总览：从 LLM 流到「可恢复页面」

```
LLM 流式 chunk
    → llm_client.astream + stream_from_llm_chunks（解析 think / message）
        → Orchestrator _emit_sse（thought / message / status → SSE）
            → state_snapshot 聚合（text / reasoning / steps / …）
                → 前端直播：增量 DOM + 执行记录
                → 入库 content.state_snapshot → 历史时光机静态还原
```

**核心原则（防回退）**

| 原则 | 说明 |
|------|------|
| **双轨分流** | 推理/思考与对用户可见正文在**协议层**分开，避免混进同一段 `message` 污染快照与 UI。 |
| **增量优先** | 流式以 chunk 消费；直播思考区禁止用整包 `innerHTML` 覆盖。 |
| **跨块安全** | `think` 等标签可被切在两次 chunk 之间 → **缓冲区 + 状态机**（不可单次 `indexOf`）。 |
| **快照一致** | `state_snapshot` 中 **`reasoning` 与 `text`** 的写入规则须与 SSE `event` 类型对齐，历史才能单独还原思考区。 |

## 二、后端：流式解析、SSE 与快照写入

### 2.1 分层职责

1. **传输层：`LLMClient.astream`**  
   OpenAI 兼容 `chat.completions.create(stream=True)` → `ChatCompletionChunk` 异步迭代。见 `gibh_agent/core/llm_client.py`。

2. **解析层：`stream_from_llm_chunks`**（推荐默认路径）  
   - **轨 A**：`delta.reasoning_content` → `("thought", {"content": ...})`。  
   - **轨 B**：对 `delta.content` 缓冲 + `in_think_tag`，在 `THINK_OPEN` / `THINK_CLOSE` 之间产出 `thought`，之外产出 `message`。  
   - 扩展块（如 `<<<SUGGESTIONS>>>`）分拣时勿打断 think 状态机。  
   见 `gibh_agent/core/stream_utils.py`（含尾部保留策略注释）。

3. **编排层：`_emit_sse`**  
   将 `(event_type, data)` 转为 SSE；**硬约束**：`thought` 只累加 **`state_snapshot["reasoning"]`**，`message` 只累加 **`state_snapshot["text"]`**，禁止交叉写入。见 `gibh_agent/core/orchestrator.py`。

4. **非流式回退**  
   `achat` + `extract_think_and_content`（`llm_client.py`）；`base_agent` 非流式路径。

### 2.2 与前端协作（协议侧）

- SSE 按 `event` 分发；`thought` / `message` / `status` 语义稳定。  
- **`[AgenticLog] ` 前缀 JSON**（技能快车道等）须独立分支解析，不得当普通步骤文案。见 `skill_agent.py` 与 `index.html` 中 `handleAgenticLog`。

### 2.3 扩展新流式能力时的检查清单

1. 新路由是否 `async for` 消费流并打首包/末包日志？  
2. 是否复用 `stream_from_llm_chunks`，避免复制 think 解析？  
3. SSE 与 `_emit_sse` 快照字段是否同步？  
4. Planner 类 JSON 是否用 `stream_and_extract_json` 或同等缓冲闭合检测？

## 三、历史快照：`state_snapshot` 与时光机

本节浓缩自 **`STATE_SNAPSHOT_AUDIT_AND_SCHEMA_DRAFT.md`**：目标是**仅用 JSON** 还原对话页（计时器、执行记录、思考区、报告建议等），**禁止存 DOM**；新字段须**向前兼容**旧历史。

### 3.1 与「思考过程」直接相关的缺口与方向

| 问题 | 影响 |
|------|------|
| `thought` 与 `message` 若都混写入 `text`、无独立 **`reasoning`** | 历史无法渲染单独「思考过程」折叠区 |
| **`duration`** 未在入库前写入快照 | 历史 footer 无「X.Xs」计时 |
| **`process_log`** 未随 status 累积 | 历史缺少「正在接收请求…」等中间态 |
| **`steps[].duration`** 未由 Executor 填充 | checklist / 报告副标题无单步耗时 |

### 3.2 推荐快照字段（在现有 `text` / `workflow` / `steps` / `report` 上增量）

| 字段 | 类型 | 说明 |
|------|------|------|
| **`reasoning`** | string | 仅思考正文，与 `text` 分离，供历史 `reasoning-zone` 还原 |
| **`duration`** | number | 本条总耗时（秒），与 done 事件一致 |
| **`process_log`** | array | `{content, state}` 与直播 process-log-zone 顺序一致 |
| **`steps[].duration`** | number | 单步耗时（秒） |

读取快照时：上述字段**一律可选**；缺省则对应 UI 不展示或回退旧逻辑，**不得白屏**。

### 3.3 历史恢复前端原则

- 骨架顺序：**reasoning-zone → process-log-zone → text-zone → cards-zone**。  
- 用 `createElement` / `appendChild` 或已有静态渲染函数填充；**禁止**对整泡或 reasoning-zone 做 `innerHTML` 全量覆盖。  
- 若 `snap.reasoning` 存在：挂载与直播一致的思考折叠块（如 `.agent-thought-block`）。

## 四、前端：思考区两条渲染路径

本节主体来自 **`FRONTEND_THOUGHT_RENDERING_REPLAY.md`**，整理为可执行规范。

### 4.1 路径划分

| 场景 | 条件 | 策略 |
|------|------|------|
| **纯文本闲聊** | `workflowData` 为空 | `agent-thought-block` + **`thought` 增量 `appendChild(TextNode)`**（打字机） |
| **结构化** | `workflowData` 非空（工作流/报告） | `updateChatBubbleWithReasoning()` → `thinking-box` 挂到 **`reasoning-container`**（修层级错位） |

纯文本路径**不得**误走 `updateChatBubbleWithReasoning()` 整段覆盖，否则会失去增量节奏并可能破坏历史结构。

### 4.2 纯文本路径：关键状态与 DOM

- **`window.currentThoughtContainer`** → `.agent-thought-block .thought-content`（`ensureThoughtBlock()` 赋值）。  
- **`thoughtBuffer`**：调试累积；**落屏仍靠每段 `thoughtText` 增量 append**。  
- **`messageBuffer`**：累积正文；`message` 时去掉 think 标签后 `updateChatBubble`。

**首次 `thought` 时 DOM（`details.agent-thought-block`）**

- `summary`：开/关图标 +「思考中…」  
- **`div.thought-content`**：增量文本唯一落点；插入父级：`reasoningContainer || messageContent`。

### 4.3 `case 'thought'` 处理要点

1. 从 payload 抽取 **`thoughtText`**（兼容 `string` / `content` / `text` / `reasoning` 等形态）。  
2. 确保 `currentThoughtContainer` 存在（必要时 `ensureThoughtBlock()`）。  
3. 若 `thoughtText` 非空：`appendChild(document.createTextNode(thoughtText))`；`scrollTop = scrollHeight`；对上层 `details` **`setAttribute('open','')`**。

### 4.4 `case 'message'` 处理要点

1. **`finalizeThoughtBlock()`**：关 `open`、换 summary 为「思考过程」。  
2. 从 `messageBuffer` 清理 think 相关标签（示例：`messageBuffer.replace(/<\/?think>/g, '')`，以线上 `index.html` 为准）。  
3. **`updateChatBubble(cleanedText)`**：只更新 **`.textZone` / `aiContentDiv`**，不碰 reasoning 区思考 DOM。

### 4.5 历史 DOM 保留

- **`clearThoughtBlock()`** 只清全局引用，**不对历史消息**做 `textContent=''`、移除 `open`、清空 `.thinking-box` 等，以便时光机可回看旧思考块。

### 4.6 复刻检查清单（前端）

1. thought 落点：`.agent-thought-block .thought-content`。  
2. 增量：`appendChild(TextNode)`，禁止 `textContent = 全量`（纯文本路径）。  
3. 滚动：`scrollTop = scrollHeight`。  
4. `details` 保持 `open`。  
5. `message` 时 `finalizeThoughtBlock` + 清标签 + 只更新正文区。  
6. 历史：清空仅全局引用，不动历史节点内容。

## 五、端到端对齐表（直播 ↔ 快照 ↔ 历史）

| 能力 | 直播（SSE） | 快照字段（目标） | 历史 UI |
|------|-------------|------------------|---------|
| 思考正文 | `thought` 事件 | `reasoning` 累积 | reasoning-zone 折叠块 |
| 回答正文 | `message` 事件 | `text` | text-zone |
| 总耗时 | `done` 带 duration | `duration` | footer timer |
| 执行记录 | `status` | `process_log` | process-log-zone |
| 工作流卡片 | `workflow` | `workflow` | cards-zone |

实现时以 **`_emit_sse` 写入规则** 与 **`handleServerEvent` 分支** 对照本表自检。

## 附录 A：外部 HTTP 工具（与流式思考并列的扩展范式）
