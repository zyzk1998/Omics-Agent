## 四联同步（变更任一处须核对其余三处）

| 链路 | 约定 |
| --- | --- |
| **`thought` SSE** | 写入快照 **`reasoning`**，不得混入 `text` |
| **`message` SSE** | 写入快照 **`text`** |
| **`status` SSE**（非 `[AgenticLog]`） | 追加 **`process_log`** 条目 `{content, state}`；**`done`** 须写 **`duration`** |
| **历史恢复** | 前端 `handleServerEvent` / `createRestoreAIMessageRowWithSnapshot` / `mergeAgentContentToSnapshot` 与上表一致 |

## 前端多任务状态机与视图穿透攻坚战

> **背景**：技能广场「使用中」会话在后台跑 SSE 时，用户切去技能广场 / 历史 / 新建会话，曾出现 **高亮死锁**（侧栏卡片底色永不消退）、**误杀流**（`abort()` 或 `innerHTML=''` 清掉活 DOM）、**连续点击拦截**等问题。  
> **终局方案**：**DOM 物理隔离 + 双轨路由（Dual-Layer Routing）**，见 `services/nginx/html/index.html` 与 `docs/持久化层构建规范.md` § 10。

### 1. 演进路径：从「逻辑切换」到「物理隔离」

| 阶段 | 做法 | 典型故障 |
|------|------|----------|
| **V0 单容器** | 仅 `#chat-main-host` / `#chatContent`；切视图时 `loadSessionAndRestore` 或 `clearChatAndReset` 清 DOM | 后台 SSE 仍在写已被销毁的节点；切回后会话空白 |
| **V1 逻辑保活** | `forceSwitchToLiveChat` 只做 `display` 切换；Immortal SSE 注册 | 与 `switchMainView('chat')` 混用，仍走同一容器；新建会话 `abortAllImmortalStreams()` 误杀技能流 |
| **V2 双轨容器（现行）** | **容器 A** `#chat-main-host`：普通闲聊 / 历史恢复；**容器 B** `#skill-live-host`：「正在使用的技能」专属活会话 | 视图切换仅 CSS 显隐；清理 / 新建只动 A；技能 SSE 只写 B |

核心状态变量（前端）：

- `_dualRouteActive`：`main` | `skill_live` — 当前可见主视图轨
- `_sseRenderHost`：`main` | `skill` — SSE / `appendMessage` 写入哪条 DOM 轨
- `_skillLiveSessionId` — 技能活会话 session，与 `currentSessionId`（主轨）解耦
- `registerImmortalController(..., { host: 'skill' })` — 标记不可被「新建会话」误 abort 的句柄

### 2. 病灶复盘

**高亮死锁**

- 旧版 `.sidebar-active-skill-wrapper.active-skill-entry` **默认带蓝色底色**，不区分「正在看技能会话」与「仅后台运行」。
- 切去技能广场后视图已变，但 CSS 仍保持「选中」外观 → 用户以为仍在前台会话。
- **修复**：底色改为仅 `.active-selected` 在 **技能活视图可见** 且存在 `_activeSkillPayload` 时启用；**呼吸灯** `.active-skill-running` 独立由 `setActiveSkillRunningState` 控制，切页不剥夺。

**误杀流**

- `clearChatAndReset` / `loadSessionAndRestore` 内调用 `abortAllImmortalStreams()` 或 `chatContent.innerHTML = ''`，把正在运行的技能 SSE 一并终止或丢失已渲染节点。
- **修复**：`abortMainChatStreamsOnly({ preserveSkill: true })`；清空范围限定 `#chatContent`；`#skillLiveContent` **禁止**在导航路由中清空或 abort。

**连续点击拦截**

- 捕获阶段委托在已处于 `#skill-live-host` 可见时仍重复触发 `forceSwitchToLiveChat`。
- **修复**：`isSkillLiveViewVisible()` 为真时 early-return，仅 `preventDefault` 不再重入。

### 3. 双轨路由入口（须遵守）

| 用户动作 | 入口 | 容器 |
|----------|------|------|
| 技能广场「使用 → 新建」 | `activateSkillLiveRoute` → `forceSwitchToLiveChat` | B |
| 侧栏「正在使用的技能」 | `forceSwitchToLiveChat` | B |
| 新建会话 / Logo 回首页 | `clearChatAndReset` / `clearMainChatOnly` | 仅清 A |
| 普通聊天 / 历史点进其他会话 | `switchMainView('chat')` / `loadSessionAndRestore` | A |
| 技能广场 / 历史 / 资产 | `switchMainView(...)` | 隐藏 A+B 中非目标 |

**禁止**：在 `switchMainView`、`showChatView`、`loadSessionAndRestore` 中对 `#skill-live-host` 执行 `innerHTML=''` 或 `abortAllImmortalStreams()`（除非用户显式「清除使用中技能」）。

### 4. 与时光机四联的关系

- 技能活会话的 **Live SSE** 仍须遵守上文四联：`thought` → `reasoning`，`message` → `text`，`status` → `process_log`。
- 历史恢复（`loadSessionAndRestore`）只恢复 **主轨** `#chatContent`；技能活会话若仍在 B 轨 DOM 中，同 sessionId 命中时走 `forceSwitchToLiveChat`，**不**重新 fetch 覆盖活 DOM。
- 变更 SSE 形态或快照写入时，除四联外须核对：双轨下 `getActiveChatContentEl()` / `.active-chat-content` 选择器是否仍指向正确容器。
