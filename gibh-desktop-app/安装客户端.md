# 桌面客户端：装好、更新、从服务器拷到本机

下面这张表是**按你这组机器写死**的地址，全文命令都按这个来。以后换了路径或 IP，用编辑器按表里每一项在全文替换即可。

| 你提供的项 | 当前写死的值 |
| :--- | :--- |
| 1. 本机 `dist-out`（从服务器拉安装包 / AppImage 落地到这里） | `D:\omics agent\gibh-desktop-app\dist-out` |
| 2. SSH（用户名 + 服务器 IP） | `ubuntu` + `192.168.32.31` → 命令里写成 `ubuntu@192.168.32.31` |
| 3. 服务器上项目根目录 | `/home/ubuntu/GIBH-AGENT-V2` |

**顺带记一笔（不用你另填）：** 工程根目录在 **`dist-out` 的上一级**，也就是 **`D:\omics agent\gibh-desktop-app`**。里面有 `package.json`，**`npm install` / `npm run dist` 必须在这个文件夹里做**，不能把 `package.json` 只拷进 `dist-out`。

**PowerShell 里用 `scp` 时**：本机路径外面用**单引号**，并且**一定要写到具体文件名**，不要写成 `"...\文件夹\"` 这种**反斜杠紧挨着结束引号**，否则会报错 `No such file or directory`。

安装包文件名里会带版本号，例如 `Omics Agent Setup 0.11.0.exe`，具体数字以同步后的 `package.json` 里的 `version` 为准。

## 从宿主机拷到你电脑的 `dist-out`（在你自己的 Windows 上操作）

说明白一点：**「迁移」是在你电脑上开终端，用网络把服务器上的文件抓下来**，不是去服务器上执行复制到你 D 盘。

宿主机路径 **`/home/ubuntu/GIBH-AGENT-V2/gibh-desktop-app/dist-out`** 里，常见是 **Linux 安装包（`.AppImage`）**，不一定有 **Windows 的 `.exe`**。  
**Windows 安装包**若已经有人放到服务器上，往往在：

`/home/ubuntu/GIBH-AGENT-V2/services/nginx/html/downloads/`

下面命令都在 **你电脑的 PowerShell** 里一行一行执行。

**1）从服务器 `gibh-desktop-app/dist-out` 拉 AppImage（Linux 用；版本号按服务器上实际文件名改）**

```powershell
scp 'ubuntu@192.168.32.31:/home/ubuntu/GIBH-AGENT-V2/gibh-desktop-app/dist-out/Omics Agent-0.11.0.AppImage' 'D:\omics agent\gibh-desktop-app\dist-out\Omics Agent-0.11.0.AppImage'
```

**2）从服务器 `downloads` 拉 Windows 安装包（`.exe`；文件名必须和服务器上的一致，去服务器上用 `ls` 看一眼再改）**

```powershell
scp 'ubuntu@192.168.32.31:/home/ubuntu/GIBH-AGENT-V2/services/nginx/html/downloads/Omics Agent Setup 1.0.0.exe' 'D:\omics agent\gibh-desktop-app\dist-out\Omics Agent Setup 1.0.0.exe'
```

第一次会问 `Are you sure you want to continue connecting`，输入 **`yes`** 回车，再输入服务器密码。

若提示没有 `scp` 命令：在 Windows **设置 → 应用 → 可选功能** 里打开 **OpenSSH 客户端**，或用 **WinSCP** 图形界面，左边连服务器，右边选你本机的 `D:\omics agent\gibh-desktop-app\dist-out`，把文件拖过去。

**只拉了 AppImage 不够：** 打 Windows 安装包时，**版本名、软件显示名**全看你电脑上的 **`package.json`**。若你 D 盘里还是旧的 `1.0.0` / `GIBH Agent Demo`，`npm run dist` 就永远出旧 exe。请先在 **本机 PowerShell** 用 `scp` 从服务器把下面四个文件**盖到你 D 盘的 `gibh-desktop-app` 根目录**（每条都写到**具体文件名**），再重新 `install` 和 `run dist`：

```powershell
scp 'ubuntu@192.168.32.31:/home/ubuntu/GIBH-AGENT-V2/gibh-desktop-app/package.json' 'D:\omics agent\gibh-desktop-app\package.json'
```

```powershell
scp 'ubuntu@192.168.32.31:/home/ubuntu/GIBH-AGENT-V2/gibh-desktop-app/package-lock.json' 'D:\omics agent\gibh-desktop-app\package-lock.json'
```

```powershell
scp 'ubuntu@192.168.32.31:/home/ubuntu/GIBH-AGENT-V2/gibh-desktop-app/main.js' 'D:\omics agent\gibh-desktop-app\main.js'
```

```powershell
scp 'ubuntu@192.168.32.31:/home/ubuntu/GIBH-AGENT-V2/gibh-desktop-app/app-icon.png' 'D:\omics agent\gibh-desktop-app\app-icon.png'
```

盖完后再执行：

```powershell
& "$env:ProgramFiles\nodejs\npm.cmd" install
```

```powershell
& "$env:ProgramFiles\nodejs\npm.cmd" run dist
```

此时应打出 **`Omics Agent Setup 0.11.0.exe`**（与服务器 `version` 一致）。

**重要：** 若你网页下载永远是旧版，多半是 **`downloads` 文件夹里的 `.exe` 没换成新的**。需要有人在 Windows 上打出新 `Omics Agent Setup x.x.x.exe` 后，**先覆盖上传到服务器的 `downloads`**，再按下面「第二步」重启 nginx；只从 `dist-out` 拉 AppImage **不能**当 Windows 安装包用。

## 第一步：在你电脑上打出 Windows 安装包

1. 用资源管理器打开你放项目的文件夹，进入里面的 **`gibh-desktop-app`** 文件夹（里面要有 `package.json` 这个文件）。

2. 在文件夹空白处按住 **Shift** 再点右键，选 **「在此处打开 PowerShell 窗口」** 或 **「在终端中打开」**。

3. 先进入当前这个文件夹（如果你打开的终端已经在 `gibh-desktop-app` 里，这条可以跳过）：

```powershell
Set-Location -LiteralPath 'D:\omics agent\gibh-desktop-app'
```

路径已按上表写死；若你换过盘符或文件夹名，只改这一条里的路径。

4. 装依赖（如果下面这条能跑通，就不用管「禁止运行脚本」）：

```powershell
& "$env:ProgramFiles\nodejs\npm.cmd" install
```

5. 打包：

```powershell
& "$env:ProgramFiles\nodejs\npm.cmd" run dist
```

这一步会自动根据 **`app-icon.png`** 在 **`build`** 文件夹里生成 **`icon.ico`**，再打出安装包；**不要删** `app-icon.png`。`build/icon.ico` 是打包时临时生成的，不用手改，一般也不用提交到 Git。

6. 打包结束后，在同一层里打开 **`dist-out`** 文件夹，里面会有一个 **`Omics Agent Setup` 后面带版本号的 `.exe`** 文件。记住它的完整文件名。

7. 把这个 `.exe` 用 U 盘、微信、WinSCP 等你习惯的方式，**复制到服务器**上项目里的这个目录里（没有就新建同名文件夹）：

`GIBH-AGENT-V2\services\nginx\html\downloads\`

（在 Linux 服务器上一般是：`.../GIBH-AGENT-V2/services/nginx/html/downloads/`）

---

## 第二步：在宿主机（服务器）上让下载页生效

用 SSH 登录到你的 Linux 服务器，进入项目根目录（路径按你实际放的改）：

```bash
cd /home/ubuntu/GIBH-AGENT-V2
```

重启一下 nginx 容器（让网站读到新的安装包）：

```bash
docker compose restart nginx
```

---

## 第三步：在你电脑上下载并安装

1. 用浏览器打开你们平时访问控制台的那个地址，后面加上 **`/downloads/`**，例如：

`http://192.168.32.31:8018/downloads/`

2. 页面上点 **「下载 Windows 安装包」**，把 `.exe` 下载到你电脑。

3. 双击下载好的 `.exe`，按提示下一步安装。如果 Windows 弹出「已阻止」，点 **「更多信息」** 再点 **「仍要运行」**。

4. 装好后在开始菜单或桌面找到 **Omics Agent** 打开即可。

---

## 如果第 4 步里 `npm` 一直红色报错

就只用 **`npm.cmd`**，不要用单独的 `npm` 两个字。上面第 4、5 步已经写成 `npm.cmd` 了。

如果提示找不到 `npm.cmd`，打开 **CMD**（不是蓝色 PowerShell），执行：

```bat
where npm
```

看显示的路径，把第 4、5 步里的 `C:\Program Files\nodejs\npm.cmd` 换成你电脑上 `where` 出来的路径。
